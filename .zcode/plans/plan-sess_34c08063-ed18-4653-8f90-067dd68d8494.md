## Plan: Publish-all + per-page unlist (visibility flag)

### Model chosen
An `is_published` boolean column on `pages`. Unlisting flips the flag to `false` but **keeps `published_config` intact**; re-listing flips it back with content unchanged — a true reversible toggle. The existing draft→published copy semantics are untouched.

### Why default `true`
Existing pages, kit-applied pages, and the `published()` factory state all assume "if it has a published_config, it's live." Defaulting `is_published = true` on the column means every page that has ever been published stays live after migration, and the existing single-page publish flow needs no change. Unlisting is an explicit, opt-out action.

### Public 404 gate
One change to `TenantPublicSiteController::show` (line 36-37):
```php
if (! $page->is_published || ! $page->published_config) {
    abort(404, 'This page is not available.');
}
```
Both conditions must hold for a page to render. An unlisted page (flag false) 404s even with content; a listed page with no published_config still 404s.
